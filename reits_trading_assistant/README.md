# REITs 交易助手

REITs 二级策略复盘工具，用于分析账户净值表现、交易行为、板块配置偏移和择时效果。

## 一、原始数据管理

所有原始文件放在 `data/raw/` 目录下。

### 必要文件（缺失则程序报错）

| 文件名 | 更新频率 | 说明 |
|--------|----------|------|
| `日报表_中诚信托-明珠76号_YYYYMMDD.xlsx` | 每日 | 含净值时间序列和账户净资产。程序**自动识别最新日期**文件。 |
| `统计分析-持仓查询-组合持仓查询*.xlsx` | 定期 | 含每日持仓市值，用于计算仓位比例。 |
| `统计分析-交易查询-交易所成交查询*.csv` | 定期 | 含所有交易明细。支持多种脏数据格式，程序自动清洗。 |
| `沪深REITs.xlsx` | 有新REITs上市时 | REITs基础信息（代码、名称、板块分类）。 |
| `history data.xlsx` | 一次性 | 历史净值和指数数据（2022-11-24 起）。已有则不需再改。 |

### 可选文件（缺失则自动降级）

| 文件名 | 说明 |
|--------|------|
| `指数.xlsx` | 指数历史数据。**优先使用 Wind 增量缓存**，仅作回退。 |
| `行情数据251231至今.xlsx` | 个股行情。**优先使用 Wind 缓存**，仅作回退。 |
| `932006closeweight.xlsx` | 中证 REITs 指数成分权重，用于配置偏移分析。 |

### 文件更新操作

**每日更新流程（典型场景）：**

1. 下载当日日报表，放入 `data/raw/`，文件名格式 `日报表_中诚信托-明珠76号_YYYYMMDD.xlsx`。
2. 旧日报表可以**保留也可以删除**，程序只读最新日期的文件。
3. 持仓和交易文件有更新时替换对应文件。
4. 运行程序，Wind 指数缓存（`data/processed/index_cache.csv`）和个股行情缓存（`data/processed/wind_prices_cache.csv`）会**自动增量更新**，无需手动操作。

**何时删缓存：**

- 需要强制全量刷新指数数据时：删除 `data/processed/index_cache.csv`。
- 需要强制全量刷新个股行情时：删除 `data/processed/wind_prices_cache.csv`。

## 二、模块架构与数据流

### 总体数据流

```
data/raw/（原始文件）
     │
     ▼
data_loader.py  ──►  data/processed/（清洗后的 CSV/Parquet）
     │
     ├──► sector_analysis.py   ──►  板块交易分布图
     ├──► trade_analysis.py    ──►  仓位/资金流向图
     ├──► timing_analysis.py   ──►  择时胜率图
     ├──► performance_analysis.py ──►  净值/超额图 + 逐日跟踪表
     ├──► allocation_analysis.py  ──►  配置偏移表
     └──► report_generator.py     ──►  Markdown 报告

data/processed/（Parquet/CSV）
     │
     ▼
dashboard.py  ──►  浏览器交互看板
```

### 各模块说明

#### `config.py` — 全局配置中心

所有路径、文件名、分析参数集中在此文件。更改账户名称、基准日期、Wind 开关等配置，只改这一个文件即可。

关键参数：
- `BASE_DATE`：净值归一化基准日（默认 `2025-12-31`）
- `ACCOUNT_NAME`：账户名称，用于自动匹配日报表文件名
- `USE_WIND_API`：是否启用 Wind 数据（默认 `True`）

#### `src/data_loader.py` — 数据加载与清洗

职责：读取所有原始文件，清洗脏数据，对齐时间序列，输出供分析模块使用的 DataFrame。

关键处理：
- **历史对齐**：将 2022 年以来的历史净值/指数与当前计算数据拼接为完整时间序列，以基准日为锚点做 scale 校准。
- **持仓数据**：按日汇总持仓市值，缺失日期用前向填充（ffill），计算仓位 = 持仓市值 / 净资产。
- **交易清洗**：处理 `="数字"` 格式、全角逗号、.SH/.SZ 后缀等脏数据。

输出到 `data/processed/`：
- `daily.csv` / `daily_master.parquet`：日频主表（净值、指数、仓位、超额收益）
- `trades_clean.csv`：清洗后的交易明细
- `holdings.csv`：持仓数据
- `performance_summary_*.parquet`：业绩指标
- `allocation_bias_*.parquet`：配置偏移数据

#### `src/wind_data_loader.py` — 行情数据获取

职责：从 Wind API 获取 REITs 个股行情和指数数据，本地缓存增量更新，Wind 不可用时自动回退到本地 Excel。

逻辑：每次运行时读取缓存 → 找最大日期 → 仅拉取新增区间 → 合并写回缓存。

#### `src/sector_analysis.py` — 板块分析

- 按板块（产业园区、保障房、高速公路等）汇总买入/卖出/净买入金额。
- 计算区间内各板块的平均涨跌幅。
- 输出气泡图：X 轴=板块涨跌幅，Y 轴=净买入，气泡大小=交易量，四象限判断操作方向是否与行情匹配。

#### `src/trade_analysis.py` — 交易行为分析

- 按日汇总净买入，以 75 分位数为阈值识别**大幅加仓（heavy_buy）**和**大幅减仓（heavy_sell）**信号。
- 计算每日仓位（持仓市值 / 净资产），标注单日变动 > 10% 的异常点。
- 输出资金流向图、仓位趋势图、净买入 vs 指数图。

#### `src/timing_analysis.py` — 择时效果分析

- 对每个 heavy_buy / heavy_sell 信号，统计操作后 **5 / 10 / 20 个交易日**的指数涨跌幅。
- 买入胜率 = 买入后指数上涨次数 / 买入总次数。
- 卖出胜率 = 卖出后指数下跌次数 / 卖出总次数。
- 在指数曲线上标注所有买卖信号点，直观展示操作时机。

#### `src/performance_analysis.py` — 业绩分析

- 计算年化收益率、年化波动率、夏普比率（无风险利率 = 十年期国债）、最大回撤、超额收益。
- 分月/分季度拆解收益归因。
- 净值 vs 指数对比图：两条线以同一基准日归一化，填充超额区域（账户跑赢时绿色，跑输时红色）。

#### `src/allocation_analysis.py` — 配置偏移分析

- 将账户持仓权重与中证 932006 指数权重对比，计算个券偏移和板块偏移。
- 正值 = 超配，负值 = 低配。

#### `src/report_generator.py` — 报告生成

- 汇总所有分析结果，生成结构化 Markdown 报告，包含核心结论、交易行为摘要、择时胜率、风险提示。

## 三、策略复盘逻辑

复盘分为三层：

### 第一层：业绩归因（我赚了多少，相对谁）

- **净值 vs 指数**：账户净值与中证 REITs 全收益指数（932047）以基准日归一化对比，填充区域体现超额收益的积累和回吐。
- **分月收益**：对比账户月度收益与指数月度收益，找出贡献正超额和拖累超额的时段。
- **关键指标**：年化收益、年化超额、夏普比率、最大回撤。

### 第二层：行为诊断（钱花在哪，操作对不对）

- **仓位管理**：仓位水位时间序列，观察加减仓节奏是否与指数趋势匹配。
- **调仓意图**：仓位变动（ppt）vs 指数，判断每次大幅调仓是顺势还是逆势。
- **择时胜率**：大幅加减仓后 5/10/20 日指数涨跌的统计，量化判断择时能力。

### 第三层：结构暴露（押注在哪，结构对不对）

- **板块操作归因**：四象限气泡图，判断每个板块的操作方向（净买入正/负）是否与该板块的涨跌幅方向一致。右上（涨了且买了）和左下（跌了且卖了）为有效操作，左上（涨了但卖了）和右下（跌了但买了）为反向操作。
- **配置偏移**：期初 vs 期末的板块超配/低配变化，分析结构暴露是否主动调整。

## 四、结果查看方式

### 方式一：交互式 Dashboard（推荐）

启动 Streamlit 看板：

```bash
PYTHONIOENCODING=utf-8 streamlit run dashboard.py
```

浏览器访问 `http://localhost:8501`。

**看板包含六张图：**

| 图表 | 内容 | 核心交互 |
|------|------|----------|
| 图一：核心趋势 | 净值 vs 指数（归一化对比） | 侧边栏选择**归一化基准日**，顶部指标卡联动更新 |
| 图二：时间归因 | 分月收益对比（账户 vs 指数） | 选择分析区间（起始日~结束日） |
| 图三：调仓意图 | 仓位变动 ppt，双轴图 | 同上 |
| 图四：持仓状态 | 仓位水位，面积图 | 同上 |
| 图五：结构暴露 | 板块配置偏移，期初 vs 期末 | 同上 |
| 图六：战术得失 | 板块操作归因，四象限气泡图 | 同上 |

**侧边栏操作说明：**
- **图一归一化基准日**：选择净值/指数的归一化起点，顶部四个指标卡（区间收益、年化、最大回撤、夏普）同步计算。
- **图二~六起止日期**：独立控制分析区间，不影响图一。

### 方式二：批量运行（生成静态报告）

```bash
PYTHONIOENCODING=utf-8 D:/install/python.exe main.py
```

输出目录 `output/YYYYMMDD_HHMMSS/` 下包含：

| 文件 | 内容 |
|------|------|
| `figures/*.png` | 所有分析图表（8 类） |
| `reports/report_*.md` | Markdown 汇总报告 |
| `trade_summary.xlsx` | 交易汇总（按日 + 按个券） |
| `timing_analysis.xlsx` | 择时胜率统计表 |
| `performance_summary.xlsx` | 业绩指标表 |
| `daily_tracking.xlsx` | 逐日净值/指数/仓位跟踪 |
| `allocation_bias.xlsx` | 个券和板块配置偏移 |

### Dashboard 数据说明

Dashboard 直接读取 `data/processed/` 下的 Parquet/CSV 文件，**无需每次重新运行 `main.py`**。
若原始数据有更新，先运行 `main.py` 刷新 processed 数据，再刷新浏览器即可。
