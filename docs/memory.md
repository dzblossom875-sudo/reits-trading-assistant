#### [数据加载] 修复交易文件读取与列映射

**问题现象**：
- 交易文件 glob 模式不匹配（文件名含连字符 `-` 而非空格 ` - `）
- 列映射被同名列覆盖：`费用结算日期` 覆盖 `业务日期`，`全价成交金额` 覆盖 `成交金额`
- 仅支持 CSV，实际文件为 XLSX 格式

**根因**：
- 模糊匹配 `"日期" in col_str` 导致取到最后出现的列
- 文件搜索模式过于严格，未考虑实际命名习惯
- 未根据扩展名选择正确的 pandas 读取函数

**解决方案**：
- 优先精确匹配：`"业务日期" in col_str` 优先于 `"日期" in col_str`
- 数量/金额列增加精确匹配：`col_str == "成交数量"` 优先于 `"成交数量" in col_str`
- 同时支持 CSV/XLSX：根据扩展名调用 `pd.read_csv()` 或 `pd.read_excel()`
- glob 模式放宽为 `"统计分析*交易查询*"` 匹配 csv/xlsx

**工作流改进**：
- 加载数据时打印列映射结果，便于快速验证字段是否正确识别
- 交易文件读取失败时返回 None，调用处需做空值保护

**避坑指南**：
- Excel 列名可能包含隐藏字符或空格，匹配前先用 `.strip()` 清洗
- 同名/相似名列需用精确匹配或列索引固定，避免顺序依赖
- Windows 中文路径 glob 需用正斜杠或 `os.path.join()` 避免转义问题

**关联文件**：
- `src/data_loader.py` - `load_trades_from_exchange_query()`

#### [业绩分析] 双列对比格式与分月逻辑修正

**问题现象**：
- 总体指标原单列表格，账户与指数指标无法直接对比
- 分月表现缺少起止日期说明，计算口径为"月内首个交易日→月末"而非"上月末→本月末"

**解决方案**：
- 总体指标改为双列格式：`指标 | 账户名称 | 指数`
- 增加"区间总收益率"与"年化收益率"区分绝对收益与年化收益
- 分月计算改为：本月收益 = (本月末净值 / 上月末净值 - 1)
- 分月表增加 `起始日`、`结束日` 列明确计算区间

**避坑指南**：
- 业绩计算需明确是绝对收益还是年化收益，避免混淆
- 分月收益若用"月内首个交易日→月末"，首月会包含跨月持有收益，导致与人工核对不一致
- DataFrame.attrs 可用于传递元数据（如阈值、基准日），但需检查属性存在性

**关联文件**：
- `src/performance_analysis.py` - `save_performance_summary()`, `calc_metrics_by_period()`

#### [交易分析] 仓位计算与 heavy_buy/sell 阈值修正

**问题现象**：
- heavy_sell 信号数为 0，卖出胜率为空
- 缺少仓位时序数据，无法生成仓位 vs 指数图
- 交易汇总缺少仓位、持仓市值、净资产字段

**根因**：
- 原阈值逻辑：`|净买入|.quantile(0.75)`，同时用于正负方向
- 实际交易以净买入为主，负值绝对值小，无法触发阈值
- 持仓数据仅读取最新截面，未加载历史日频数据

**解决方案**：
- 阈值分离计算：
  - `buy_threshold = 正净买入的 75 分位数`
  - `sell_threshold = 负净卖出的 75 分位数绝对值`
- 新增 `load_holdings_timeseries()` 读取全部日期持仓市值
- 新增 `plot_position_vs_index()` 绘制仓位比例 vs 指数双轴图
- 交易汇总增加仓位(%) = 持仓市值 / 净资产

**避坑指南**：
- 多空信号阈值应分别计算，不能简单用绝对值对称处理
- 持仓市值需从原始文件读取时序（按日期 groupby sum），截面数据无法计算历史仓位
- Excel 日度汇总应按日期降序排列，方便查看最新数据

**关联文件**：
- `src/trade_analysis.py` - `summarize_trades()`, `plot_position_vs_index()`, `save_trade_summary()`
- `src/data_loader.py` - `load_holdings_timeseries()`
- `main.py` - 调用链传递 holdings_daily 和 net_assets

#### [择时分析] 卖出胜率计算修复

**问题现象**：
- `timing_analysis.xlsx` 卖出胜率为空或 N/A

**根因**：
- 原阈值逻辑导致 `heavy_sell` 信号数为 0，无样本计算胜率

**解决方案**：
- 使用分离阈值后，`heavy_sell` 天数从 0 → 12 天
- 卖出胜率 = (卖出后指数下跌次数 / 总卖出信号数)

**避坑指南**：
- 胜率统计需检查样本数是否为 0，避免除零或空值
- 择时效果评估应区分买卖方向，单边市场下某方向信号可能极少

**关联文件**：
- `src/timing_analysis.py` - `analyze_timing()`
- `src/trade_analysis.py` - 阈值计算逻辑

## 🔧 通用工具函数

### 数据清洗

```python
# 清洗 "=" 包裹的数字（Excel 导出常见格式）
def clean_number(val):
    s = str(val).strip()
    if s.startswith('="') and s.endswith('"'):
        s = s[2:-1]
    return pd.to_numeric(s.replace(',', '').replace('\uff0c', ''), errors='coerce')

# 清洗证券代码（去 .SH/.SZ，补零到6位）
def clean_code(val):
    import re
    s = str(val).strip()
    m = re.search(r'(\d+)', s)
    if m:
        return m.group(1).zfill(6)
    return None
```

## 📊 计算口径说明

### 收益率计算

| 指标 | 公式 | 说明 |
|------|------|------|
| 区间总收益率 | (期末 / 期初 - 1) × 100% | 绝对收益，非年化 |
| 年化收益率 | (1 + 总收益)^(365/天数) - 1 | 按自然日年化 |
| 年化波动率 | 日收益率.std() × √252 | 交易天数按252天 |
| 最大回撤 | max((峰值 - 当前)/峰值) | 滚动计算，取最负值 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动 | rf 默认 2% |

### 分月收益

- **起始日**：上月最后一个交易日净值（或本月首个交易日，若上月无数据）
- **结束日**：本月最后一个交易日净值
- **计算**：(结束日净值 / 起始日净值 - 1) × 100%

### Heavy Buy / Sell 定义

- **Heavy Buy**：日净买入金额 > 正净买入的 75 分位数
- **Heavy Sell**：日净卖出金额 < 负净卖出的 75 分位数（绝对值）

### 仓位计算

- **持仓市值**：当日所有 REITs 持仓市值加总（来自持仓查询文件）
- **净资产**：来自日报表"净资产（市值）"列
- **仓位比例**：持仓市值 / 净资产 × 100%
